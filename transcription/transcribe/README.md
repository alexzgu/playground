# Booklet transcription pipeline

Automates the page-by-page transcription of `Bayesian-Booklet-Fa2025.pdf` (PDF pages 23–195; pages 1–22 were transcribed by hand in an earlier session and are preserved as-is). Each page image is transcribed by Claude (`claude-opus-4-8`) with an explicit handwriting hunt, every quantitative result is machine-checked with SymPy, and pages are stitched into the chapter files at the repo root.

## Why it's built this way
- **Handwriting first.** The instructor's annotations (strikethrough corrections, margin notes, scratch derivations) are the most valuable and the easiest content to miss. The model must output a handwriting inventory *before* transcribing, and declare explicitly when a page has none. Red-ink marks (the PDF annotation layer) are additionally detected by color (`detect_annotations.py`) and fed in as magnified crops, so they can't be overlooked; dark-pen marks rely on the prompted hunt.
- **Verification is code, not vibes.** The transcriber leaves `> ⏳ VERIFY[ck]` placeholders and emits a SymPy check per quantitative result; `verify_claims.py` executes the checks (subprocess, import-guarded, 90 s timeout) and stamps `> ✔ Verified` / `> ⚠ Check FAILED` into the transcript plus rows into `verification-log.md`. A failed check means *either* a booklet error or a transcription error — both worth human eyes.
- **Nothing hardcodes the booklet page offset.** The model reads the printed page number off each page (there is a suspected unnumbered handwritten insert around PDF 161–162 that shifts numbering); the assembler groups pages into chapters by those detected numbers.

## Run it

```bash
python3 transcribe/detect_annotations.py     # once — red-ink regions + crops
python3 transcribe/transcribe.py             # all pending pages (resumable)
python3 transcribe/verify_claims.py          # run SymPy checks, stamp results
python3 transcribe/assemble_chapters.py      # rebuild chNN files + README status
```

Useful variations:

```bash
python3 transcribe/transcribe.py --pages 23-39        # one chapter's worth
python3 transcribe/transcribe.py --pages 161 --force  # redo a page
python3 transcribe/transcribe.py --backend api --workers 6   # Anthropic API backend
```

## Backends
- **cli** (default): runs headless `claude -p` per page using your existing Claude Code login — zero setup, no API key; the model reads images with its Read tool. ~1–3 min/page, 3 workers by default. Counts against your Claude plan usage.
- **api**: direct Anthropic API with base64 images + adaptive thinking. Faster and parallelizes harder, but needs credentials (`ANTHROPIC_API_KEY` or `ant auth login`). Rough cost at Opus 4.8 rates ($5/$25 per MTok): ~5–6k input + ~2–3k output tokens/page ≈ $0.08–0.12/page, so roughly **$15–20 for all 173 pages** (about half that via the Message Batches API, which this script doesn't use — see `python3 transcribe/transcribe.py --help`).

## State & outputs
- `out/manifest.json` — per-page status (`transcribed` → `verified`/`verified-with-flags`), detected booklet number, handwriting-mark count, claim counts. Delete an entry (or use `--force`) to redo a page.
- `out/raw/p-NNN.raw.md` — full model output (handwriting inventory + transcript + claims).
- `out/pages/p-NNN.md` — the transcript that goes into chapter files.
- `out/verify-results.json` — verdict + stderr tail for every check.
- Chapter files + `verification-log.md` + README status at the repo root are regenerated by the last two stages (pages 1–22 content and log rows are preserved).

## Reviewing quality
Grep the raw files for `⚠` and for `hw=0` pages in the manifest that you'd expect to carry annotations; spot-check those against `pages/p-NNN.jpg`. Failed checks are listed at the bottom of `verification-log.md`.
