# Chapter 11 — METROPOLIS-HASTINGS SAMPLER

The Metropolis-Hastings sampler was developed by Metropolis, Rosenbluth, Rosenbluth, Teller, and Teller(1953). It was generalized by Hastings(1970).

(1) Tierney, L.(1994), Markov chains for exploring posterior distributions(with discussions), Annals of statistics, 22, 1701-1762.

(2) Chib, S. and Greenberg, E.(1995), uender-standing [sic] the Metropolis-Hastings algorithm, The American Statistician, 49, 327-335.

**11.1 General Discussion**

*[margin note, right, with a long curved lead-line drawn down to the $\alpha(x,y)$ equation below: "Markov chain possesses a stationary distribution i.e. time invariant (equilibrium probability distribution)"]*

There are three components in the M-H sampler (More powerful than Accept-Rejection Algorithm).

(1) Target density

This is the density we need to draw a sampler from, and we denote it by $\pi(x)$. We do not need to know the normalization constant, only the kernel is required.

(2) Candidate-generating density (proposal)

The process is at $x$, and we generate value $y$ from the proposal density $q(x,y)$ where $\int q(x,y)dy = 1$. Note that $q(x,y)$ does not have to depend on $x$.

(3) Probability of move(jumping probability)

This is the probability that the process(chain) moves from $x$ to $y$, and is denoted by $\alpha(x,y)$. For the M-H sampler

$$\alpha(x,y) = \min\left\{1, \frac{\pi(y)q(y,x)}{\pi(x)q(x,y)}\right\}$$

*[margin note, right of the display: "detailed balance"]*

> ⏳ VERIFY[c1]: with this $\alpha$, the chain satisfies detailed balance, $\pi(x)q(x,y)\alpha(x,y) = \pi(y)q(y,x)\alpha(y,x)$, both sides equalling $\min\{\pi(x)q(x,y),\,\pi(y)q(y,x)\}$.

ensures that the chain is reversible. *[the word "reversible" is hand-underlined]*

What is the sampler?

1. Start with $x_0$.

2. Generate $x_1$ from $q(x_0, x_1)$.

3. Compute $\alpha(x_0, x_1) = \min\left\{1, \frac{\pi(x_1)q(x_1,x_0)}{\pi(x_0)q(x_0,x_1)}\right\}$, jumping/accept probability

CLAIM c1: The M-H acceptance probability $\alpha(x,y)=\min\{1,\ \pi(y)q(y,x)/(\pi(x)q(x,y))\}$ satisfies detailed balance: $\pi(x)q(x,y)\alpha(x,y) = \pi(y)q(y,x)\alpha(y,x) = \min\{\pi(x)q(x,y),\ \pi(y)q(y,x)\}$.

```python
from sympy import symbols, Min, Abs, Rational, simplify

# a = pi(x)q(x,y), b = pi(y)q(y,x): the two unnormalized flows (positive).
# Case-split on their order so the Min is decidable: b = a + d covers
# b >= a, swapping the arguments covers a >= b. d > 0; equality separately.
# Min is rewritten via Abs so SymPy can resolve it under the sign assumptions.
a, d = symbols('a d', positive=True)

def balance_ok(flow_xy, flow_yx):
    axy = Min(1, flow_yx / flow_xy)     # alpha(x, y)
    ayx = Min(1, flow_xy / flow_yx)     # alpha(y, x)
    lhs = flow_xy * axy
    rhs = flow_yx * ayx
    both = Min(flow_xy, flow_yx)
    assert simplify((lhs - rhs).rewrite(Abs)) == 0
    assert simplify((lhs - both).rewrite(Abs)) == 0
    assert simplify((rhs - both).rewrite(Abs)) == 0

balance_ok(a, a + d)   # pi(y)q(y,x) >= pi(x)q(x,y)
balance_ok(a + d, a)   # pi(x)q(x,y) >= pi(y)q(y,x)
balance_ok(a, a)       # equal flows

# alpha is a genuine probability: 0 < alpha <= 1 in both directions.
# Exact-rational spot checks across both branches of the min.
px, py, qxy, qyx = symbols('px py qxy qyx', positive=True)
alpha_xy = Min(1, (py * qyx) / (px * qxy))   # alpha(x, y)
alpha_yx = Min(1, (px * qxy) / (py * qyx))   # alpha(y, x)
for av in [Rational(1, 7), Rational(3, 2), Rational(5, 1)]:
    for bv in [Rational(2, 9), Rational(1, 1), Rational(11, 4)]:
        for u in [Rational(1, 3), Rational(4, 5)]:
            for v in [Rational(7, 2), Rational(1, 6)]:
                sub = {px: av, py: bv, qxy: u, qyx: v}
                axy = alpha_xy.subs(sub)
                ayx = alpha_yx.subs(sub)
                assert 0 < axy <= 1
                assert 0 < ayx <= 1
                assert av * u * axy == bv * v * ayx
                assert av * u * axy == min(av * u, bv * v)
```
