# The Bayesian Spine

**Statistical and machine learning, rebuilt from one principle — in 27 modules of runnable, verified Python.**

The whole course is four lines:

> 1. A model is a joint distribution over everything — knowns and unknowns.
> 2. Inference is conditioning: p(unknowns | knowns).
> 3. Prediction is marginalization: p(new | knowns) = ∫ p(new | unknowns) p(unknowns | knowns).
> 4. A decision minimizes posterior expected loss.

Everything else — maximum likelihood, ridge, confidence intervals, cross-entropy training, Kalman filters, A/B tests, deep ensembles — is a special case, an approximation, or an audit of these four lines. The course proves that claim rather than asserting it: MLE = flat-prior MAP (module 04), ridge = a Gaussian prior *to machine precision* (00, 14), James–Stein = an empirical-Bayes posterior mean (08), the Kalman gain = the conjugate shrinkage weight *byte-for-byte* (21), cross-entropy = categorical maximum likelihood (15, 25). Where the paradigms genuinely disagree — stopping rules, Lindley's paradox, conditional coverage — both sides' numbers are printed and the disagreement is named, not spun. The closing stance is **one calculus, two audits**: Bayesian conditioning for post-data belief, sampling-distribution checks for pre-data guarantees.

## Who this is for

A reader with a CS/applied-math background, working Python (numpy/scipy/pandas/sklearn), undergraduate exposure to ML/DL, and a passing acquaintance with Casella–Berger-style statistics — who wants the *judgment* that normally takes years: what breaks, when, why, and which approximations are safe. See [BRIEF.md](BRIEF.md) for the full design goals.

## How to use it

- **Read in order** (00 → 26). Later modules cite earlier printed numbers and reuse earlier code patterns; [modules/SPINE-INDEX.md](modules/SPINE-INDEX.md) is the one-page-per-module recap if you need to rejoin after a break.
- **Play the Predict beats honestly.** Marquee demos are staged *Setup → Predict → Run → Reconcile*: commit to a number before running. The course is engineered around being caught wrong — that is where the intuition reorganizes.
- **Run the code.** Every module's blocks execute in order, deterministically, in ~5–120 s:
  ```bash
  python tools/run_module.py modules/05-conjugate-updating.md
  ```
  Every number quoted in prose is printed by a block (machine-checked); every figure is generated by module code.
- **Do the exercises** (3–6 per module, solutions in collapsible blocks — each Reconcile explains *why* the naive guess missed, not just the answer).

**Environment.** Python ≥3.10 with `numpy scipy matplotlib pandas scikit-learn statsmodels arviz numpyro jax`. NumPyro is the course's probabilistic-programming layer (from-scratch numpy is the primary vehicle; the smoke-tested API idioms live in [tools/ppl_idioms.py](tools/ppl_idioms.py)). No R, no internet, no GPU needed; everything runs on 2 CPUs.

## Five experiences the course is built around

1. **The donut (12).** Most of a 1000-dimensional Gaussian's mass lives in a thin shell at radius √d ≈ 31.6; the mode holds essentially none of it.
2. **The overconfident MLE (05).** Two conversions in two impressions: the MLE says the next converts with certainty; the posterior predictive says 0.75.
3. **Same data, opposite verdicts (04).** Nine successes in twelve trials: p = 0.073 or p = 0.033 depending on the experimenter's *intention* — with an identical likelihood and identical posterior.
4. **Stein's paradox (08).** For three or more unrelated quantities, shrinking every estimate toward a common center beats the sample means everywhere — and the shrinkage rule *is* an empirical-Bayes posterior mean.
5. **Occam for free (17).** The marginal likelihood picks the true polynomial degree while training error keeps falling — complexity control by integration, no cross-validation required.

## What "verified" means here
