# Curriculum Brief — authoritative statement of goals and constraints

Every author, reviewer, and verifier agent reads this first. Deviations from this brief are defects.

## The learner

- BS in computer science + applied mathematics. Comfortable with linear algebra, multivariable calculus, undergraduate probability, algorithms.
- Prior exposure (working knowledge, not mastery): AI/ML/DL fundamentals (supervised learning, loss minimization, backprop, common architectures), some generative AI (VAEs/diffusion/LLMs at a survey level), basic stochastic calculus (Brownian motion, Itô integral — Lawler-style intro), basic experimental design.
- Surface-level pass through Casella–Berger ch. 1–10; skimmed the MA 556 Bayesian booklet. Treat these as "seen the words, not internalized the ideas" — recap fast, but never assume a result is already intuitive.
- Python-fluent including numpy/scipy/pandas/matplotlib/sklearn. **No R anywhere — not even in asides.**

## The goal

Relearn statistical and machine learning as **one coherent subject** built on a Bayesian foundation:

> A model is a joint distribution over everything — knowns and unknowns.
> Inference is conditioning on what you observed.
> Prediction is the posterior predictive.
> A decision is whatever minimizes expected loss under the posterior.

- **Do not tribalize the Bayesian/frequentist split.** The operative distinction is bookkeeping: which quantities you model as random (unknown to you) versus fixed (known, or conditioned on). Frequentist ideas are taught seriously and framed as special cases, approximations, or audit tools for Bayesian procedures: MLE = MAP under a flat prior; ridge/lasso = Gaussian/Laplace priors; confidence ≈ credible under Bernstein–von Mises; complete-class theorems; sampling-distribution calibration as a quality check on Bayesian models. Where the paradigms genuinely disagree (stopping rules, conditioning vs. unconditional guarantees, Lindley's paradox), say so honestly and concretely — no straw men in either direction.
- MCMC covered deeply (Metropolis–Hastings, Gibbs, HMC/NUTS) but computation is not the whole course: exact conjugate analysis, Laplace approximation, EM, and variational inference get real treatment.
- Decision theory and experimental design are first-class topics, not appendices.
- Modern ML connections (regularization, deep learning, generative models) are integrated throughout, not ghettoized in a final chapter.

## Quality bar (non-negotiable)

1. **Intuition first, rigor behind it.** Every claim is derived, simulated, or cited — ideally derived *and* simulated. "It can be shown" without a path is a defect.
2. **Concrete over abstract.** Real numbers, fully worked examples, runnable Python whose printed output the prose quotes. When prose states a number, a code block must print that number.
3. **Predict-then-verify.** Exercises and demos ask the learner to predict an outcome before running the simulation that settles it.
4. **No fluff.** No throat-clearing introductions, no motivational padding, no "in this chapter we will". Dense but readable. Concise explanations, clear examples.
5. **Self-contained and reproducible.** All code runs offline with fixed seeds; every figure is generated by module code; no dataset downloads (synthesize data or commit small CSVs).
6. **Compression of experience.** The target takeaway is the kind of judgment that normally takes years: what breaks, when, why; which approximations are safe; what practitioners actually do and why.

## Deliverable

`curriculum/` — numbered markdown modules (target 2,500–5,000 words of prose + 4–10 runnable Python blocks + exercises with solutions each), a README syllabus, figures generated into `curriculum/figures/`, all code verified by `curriculum/tools/run_module.py`.
